解决 CLIENT DOES NOT SUPPORT AUTHENTICATION

发表于 2022-06-24 更新于 2022-06-25 分类于 mysql
mysql 升级mysql8.0.29 navicat 链接报 CLIENT DOES NOT SUPPORT AUTHENTICATION 问题

遇到的问题

刚安装完mysql8.0.11，连接时报错：
在navicat中报错client does not support authentication

问题原因

在8.0+中使用指定使用caching_sha2_password创建用户账号密码时，本地navicat的版本暂时不支持，从而导致连接认证失败。

解决办法

修改用户的加密方式 或者升级内核
为了以防出现其他问题, 选择修改用户的加密方式


执行失败的, 请查看下root用户的host 是否为localhost
(Host列指定了允许用户登录所使用的IP，比如user=root Host=192.168.1.1。这里的意思就是说root用户只能通过192.168.1.1的客户端去访问。
而%是个通配符，如果Host=192.168.1.%，那么就表示只要是IP地址前缀为“192.168.1.”的客户端都可以连接。如果Host=%，表示所有IP都有连接权限。)
如果不是,改为对应的host值即可


如果还不能执行成功, 检查语句的标点符号
